After connecting github to heroku we need to
  1. Go to core folder and in the settings files you need to set
    • STATIC_ROOT = os.path.join(BASE_DIR, 'staticfiles')
    • STATICFILES_DIRS = [os.path.join(BASE_DIR, "static")]
    • import os
  2. Having your venv activated AND INSIDE YOUR REPOSITORY, you will run in the terminal pip freeze > requirements.txt which will create a .txt file in the project's folder outside any app or the core. This .txt will contain the packages that you need to run this projects
  3. Now you need to create inside the core Procfile without any file extension. These will be some parameters that we will pass on to Heroku. Here we are going to define the WSGI server
  4. We need to install pip install gunicorn which is going to be the WSGI server that is going to hepl process our HTTP requests. REMEMBER TO UPDATE THE requirements
  5. In Procfile you will then type web: gunicorn core.wsgi being core the name of our core folder of this project
  6. Create a file called runtime.txt in which you will write python-3.8.5 the python version you want to run MAYBE THIS ISNT REQUIRED
  7. Dont forget to create the static and staticfiles folders with txt.txt in them
  8. commit to github
  9. Go to Heroku and now you can lcick on Deploy Branch
  10. You wont be able to see the project, for this you need to specify in settings.py that ALLOWED_APPS = [".herokuapp.com","127.0.0.1:8000"]
